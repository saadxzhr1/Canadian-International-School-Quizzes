STEP 01

Create Your Sprites

🧺
PLAYER (BASKET)
Draw a simple basket or bowl shape, or use the bucket sprite from the library. Position at bottom centre. Controlled by arrow keys.
☄️
METEOR
The enemy. Falls from a random x at the top, resets when it reaches the bottom or hits the player.
💎
GEM
The reward. Falls like the meteor but catching it adds 1 to the score.
🏁
GAME OVER
Text sprite. Hidden during play, appears when the game ends with the final score.

STEP 03

Build the Scripts

PLAYER
Script 1 — Setup
Runs once at the start. Resets both variables and puts the player in position. Always build this first.
🚩 when green flag clicked
setscoreto0
setlivesto3
go to x:0y:-150// bottom centre
show
PLAYER
Script 2 — Movement
A forever loop checks the keyboard every frame. Move left or right depending on which arrow is held.
🚩 when green flag clicked
forever
ifkey [← left arrow] pressed?then
change x by-12
end
ifkey [→ right arrow] pressed?then
change x by12
end
end
💡
Tip: Want to stop the player from going off-screen? Add a third if after both: check if x position > 200 then set x to 200. Same for the left side with -200. Try it as a bonus.
METEOR
Script 3 — Meteor Falls
The meteor loops forever. Each loop: jump to a random x at the top, then glide down to the bottom. The glide time uses a formula based on score — the more gems you catch, the faster the meteor falls. If it touches the player on the way down, lose a life and reset immediately.
🚩 when green flag clicked
forever
go to x: pick random-200to200 y:180
glide (2–(score÷10)) secs to x:x positiony:-180 // score 0→2s, score 10→1s, score 20→0s… add a max below!
iftouching [Player]?then
changelivesby-1
end
// loop back to top regardless — resets automatically
end
💡
How the formula works: At score 0 → glide takes 2 seconds (slow). At score 5 → 1.5 seconds. At score 10 → 1 second. The speed rises smoothly as you score more, not in sudden jumps. Use the – and ÷ blocks from the Operators tab to build it.
⚠️
Add a minimum! At score 20 the formula gives 0 seconds — instant teleport. Wrap the whole glide time in a max ( … ) (0.5) operator so it never goes below 0.5 seconds. Find max in the Operators tab.
GEM
Script 4 — Gem Falls
Exactly the same structure as the meteor, but touching the player means +1 score instead of losing a life. Build this by copying the meteor script and changing two things.
🚩 when green flag clicked
forever
go to x: pick random-200to200 y:180
glide(2.5–(score÷10))secs to x:x positiony:-180// gems always 0.5s slower than meteor
iftouching [Player]?then
changescoreby1
end
end
💡
Two things to change vs. meteor: (1) the glide time — make gems slightly slower so they're easier to catch. (2) The result of touching — change score by 1 instead of change lives by -1.
PLAYER
Script 5 — Watch for Game Over
A third script on the Player runs alongside movement. It watches the lives variable and as soon as it hits 0, broadcasts "game over" and stops everything.
🚩 when green flag clicked
repeat untillives ≤ 0
wait0.1seconds// checks 10× per second
end
broadcastgame over
stopall
💡
Why stop all? It freezes every script in the project instantly — the meteor stops falling, the gem stops gliding, movement stops. The Game Over sprite still gets to appear because its script is triggered by the broadcast, which fires just before the stop.
GAME OVER
Script 6 — Show the Result
Hidden at the start. Appears the moment it receives the broadcast. Uses join to display the score inside the message.
🚩 when green flag clicked
hide
📨 when I receivegame over
show
say join "Game Over! Score: " score
💡
The join block (inside Operators) glues two pieces together. join "Game Over! Score: " score produces something like Game Over! Score: 5. No maths needed — it just connects text and a variable.
STEP 04

Test & Debug

🐛 Three bugs to find

BUG 1
The meteor falls but lives never go down even when it clearly lands on the player. Which block detects contact, and what name must it match exactly?
BUG 2
The game never ends — even at 0 lives the game keeps going. Script 5 is supposed to stop it. What condition might be wrong: lives = 0 vs lives ≤ 0, and why does it matter?
BUG 3
The Game Over sprite appears immediately when the green flag is clicked. Which block is missing from its setup script?
STEP 05

Go Further

⭐ Easy

Speed Cap

The formula 2 – (score ÷ 10) hits 0 at score 20. Use the max operator to clamp the glide time to a minimum of 0.5 seconds so it never breaks.

+ operators · max
⭐⭐ Medium

Mid-Fall Detection

Replace the glide block with a repeat until y < -180 loop that moves the meteor down step by step. Now collision is checked continuously during the fall.

+ loop · change y
⭐⭐ Medium

Second Meteor

Duplicate the meteor sprite. Give it a different fall speed and starting delay (wait 1 second before its loop). Now two meteors fall at once.

+ duplicate · timing
⭐⭐⭐ Hard

Life Flash

When the player gets hit, make it flash by switching between two costumes 3 times. During the flash, disable collision so the player can't lose two lives at once.

+ costumes · flag variable
