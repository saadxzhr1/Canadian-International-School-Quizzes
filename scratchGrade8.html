STEP 02

Create These Variables

Go to Variables → Make a Variable. Create all four. Set them to apply to all sprites.

Variable name
Starts at
What it does
score
0
Increases by 1 each time the player catches a gem
lives
3
Decreases by 1 when a slime touches the player. Game over at 0.
speed
3
Controls how fast slimes and gems fall. Increases every 10 seconds.
playing
1
1 = game is running, 0 = game over. Used to stop clone loops cleanly.
💡
Why "playing"? Without this variable, clones keep spawning even after game over. Setting playing = 0 is a signal for every loop in the game to stop cleanly. This is a real technique used in game dev.
STEP 03

Build the Scripts

PLAYER
Script 1 — Game Setup
runs once at start
When the green flag is clicked, we reset every variable to its starting value and position the player at the bottom centre. This script must run before anything else — it's the game's "reset button."
🚩 when green flag clicked
setscoreto0
setlivesto3
setspeedto3
setplayingto1
go to x:0y:-150 // bottom centre of screen
PLAYER
Script 2 — Player Movement
forever loop
A forever loop checks the keyboard every single frame. If left arrow is held, move left. If right arrow is held, move right. The if on edge, bounce is replaced by clamping — we use an if to prevent the player from going off screen.
🚩 when green flag clicked
forever
if key [left arrow] pressed? then
change x by-12
end
if key [right arrow] pressed? then
change x by12
end
if x position >200 then
set x to200// wall on right
end
if x position <-200 then
set x to-200// wall on left
end
end
💡
Why check x position manually? Scratch's "if on edge bounce" makes sprites flip direction — ugly for a player. Clamping the x value between -200 and 200 keeps the player inside without any flip.
PLAYER
Script 3 — Difficulty Ramp
new concept: wait + loop
Every 10 seconds, speed increases by 1. This script runs alongside the others at the same time (Scratch runs all green-flag scripts at once). It stops itself when playing = 0.
🚩 when green flag clicked
repeat untilplaying = 0
wait10seconds
changespeedby1
end
SLIME
Script 4 — Slime Spawner
new concept: clones
The Slime sprite itself hides at the start — it's just a factory. Every 1.5 seconds it creates a clone of itself. Each clone will run independently (Script 5). The original Slime is never seen.
🚩 when green flag clicked
hide
repeat untilplaying = 0
create clone ofmyself
wait (1.5–0.1×speed) seconds // faster speed = shorter wait
end
⚠️
Wait formula: You can simplify this — just use wait 1 second to start. Once the game works, replace it with wait (1.5 – (speed × 0.1)) seconds for a proper ramp. Make sure this value never reaches 0 or negative!
SLIME
Script 5 — Slime Clone Behaviour
clone logic
This script runs on each clone as soon as it's created. The clone appears at a random x position at the top, shows itself, then falls. If it touches the player → lose a life, delete clone. If it reaches the bottom → just delete clone.
⬜ when I start as a clone
go to x: pick random-200to200 y:180
show
repeat untily position < -180
change y by-speed
iftouching [Player]?then
changelivesby-1
delete this clone
end
end// reached bottom
delete this clone
GEM
Script 6 — Gem Spawner + Clone
same pattern as Slime
Gems work the same way as slimes, but touching the player gives +1 score instead of losing a life. Build the spawner and clone scripts using the Slime as a model — the only differences are the touching condition and what happens.
🚩 when green flag clicked ← same as Slime Script 4
⬜ when I start as a clone
go to x:pick random-200to200y:180
show
repeat untily position < -180
change y by-speed
iftouching [Player]?then
changescoreby1
delete this clone
end
end
delete this clone
💡
Make gems rarer: Give gems a longer wait between spawns than slimes (e.g. wait 3 seconds instead of 1.5). This makes catching them feel rewarding.
PLAYER
Script 7 — Lives Monitor & Game Over
new concept: broadcast
A dedicated script watches the lives variable. The moment it hits 0, it sets playing = 0 (stopping all loops), then broadcasts "game over" to tell every sprite the game has ended.
🚩 when green flag clicked
repeat untillives ≤ 0
wait0.1seconds// checks 10x per second
end
setplayingto0
broadcastgame over
stopother scripts in sprite
GAME OVER
Script 8 — Game Over Screen
receive broadcast
The Game Over sprite is hidden at game start. When it receives the "game over" broadcast, it appears and displays the final score using the say block.
🚩 when green flag clicked
hide
📨 when I receivegame over
show
say join "GAME OVER! Score: " join score
💡
The join block (in Operators) lets you combine text and a variable into one message. join "Score: " score will display something like Score: 7.
STEP 04

Test & Debug

🐛 Common Bugs — Can You Find the Fix?

BUG 1
Slimes appear but don't move. The clone script runs but the y coordinate never changes. What block is probably missing or wrong?
BUG 2
The game never ends even when you get hit many times. Lives keeps going negative (-1, -2, -3…). Which script is supposed to stop this, and what condition might be wrong?
BUG 3
After game over, new slimes keep falling even though broadcasting "game over." What variable is supposed to stop the spawner loops, and was it set correctly?
BUG 4
The player walks off the screen to the left. The wall on the right works, but not the left. Look at Script 2 — what is likely wrong with the left wall condition?
STEP 05

Go Further — Extensions

⭐ Easy

High Score

Create a high score variable. At game over, check if score > high score, and if so, update it. The high score survives between rounds.

+ if/else · variables
⭐⭐ Medium

Costume Flash

When the player gets hit, make it flash red 3 times using costume switching and a short loop. Add brief invincibility during the flash.

+ costumes · loops · timing
⭐⭐ Medium

Power-Up Bomb

Add a third falling object — a bomb. If caught, it deletes all current slime clones on screen. Hint: use a broadcast that slimes listen for.

+ broadcast · delete clone
⭐⭐⭐ Hard

Restart Button

Add a button that resets all variables and restarts the game without pressing the green flag. It must also delete all existing clones. This is trickier than it sounds.

+ broadcast · stop scripts
⭐⭐⭐ Hard

Slime Boss

Every time the score is a multiple of 10, spawn a giant slime (larger size, half the fall speed, 3× the size). Use operators to detect multiples: score mod 10 = 0.

+ modulo operator · size · conditions
🔥 Ultimate

Two-Player Mode

Add a second player controlled by W/A/S/D keys. Each player has their own lives. The game ends when both players reach 0 lives. Display both scores at game over.

+ multi-sprite · W/A/S/D keys
