Exercise 01
01

The Shrinking Escape

Size control · Scene transitions · Precise timing
Intermediate

🎬 Scene Plan

Scene 1
The Forest
Character starts full size → shrinks → disappears
→
Scene 2
The Cave
Character appears tiny → grows back → walks to centre
→
Scene 3
Safe Place
Character bounces (loop) to celebrate escape
🎯
Goal: A character shrinks small enough to "slip through" to a new scene, reappears tiny, grows back to full size, then celebrates. The challenge is controlling timing so the shrinking finishes before the scene changes, and the growing looks smooth on the other side.
Character
🧒 Hero (you choose)
Main character. Place at left of Scene 1. Appears in all 3 scenes.
📋 Build Instructions
1
Create 3 scenes. Add your character to each scene, positioned where you want them to appear in that scene. Set the character in Scene 2 and 3 to smallest size at the start (you'll grow them with blocks).
2
Scene 1 script: The character must shrink 4 times with waits between each shrink, then change scene.
Character – Scene 1
▶ Start on Green Flag
→ Move Right 2 ⏱ Wait 1 ⬇ Shrink ⏱ Wait 1 ⬇ Shrink ⏱ Wait 1 ⬇ Shrink ⏱ Wait 1 ⬇ Shrink
🎬 Change Scene → ⬛ End
3
Scene 2 script: Start from hidden/tiny. Grow 4 times with waits, then walk to centre.
Character – Scene 2
▶ Start on Scene Begin
⬆ Grow ⏱ Wait 1 ⬆ Grow ⏱ Wait 1 ⬆ Grow ⏱ Wait 1 ⬆ Grow
→ Move Right 3 🎬 Change Scene → ⬛ End
4
Scene 3 script: Use a 🔁 Repeat loop to make the character jump up and down 3 times to celebrate. Do this with alternating Move Up → Wait → Move Down → Wait inside the loop.
⚠️
Common mistake: Using "Start on Green Flag" in Scene 2 won't work — the green flag only fires once at the very beginning. You must use "Start on Scene Begin" (the orange page trigger) for scenes 2 and 3.
💡
Think about it: What happens if you remove all the Wait blocks? Does the animation still look like shrinking? Why or why not?
⭐
Extension: Add a second character in Scene 1 — a monster chasing the hero. The monster should walk towards the hero and reach the hero's starting position right as the hero disappears. You'll need to calculate how many Move blocks the monster needs based on how far away it starts.
Exercise 02
02

The Synchronized Race

Messages · Multi-character coordination · Staggered timing
Intermediate+

🎬 Scene Plan

Scene 1
The Starting Line
3 characters wait, then race on a message signal
→
Scene 2
The Finish Line
Winner grows big · Others stay small · Podium effect
🎯
Goal: Three racers start at different speeds (controlled by different numbers in their Move blocks). A referee character sends the start signal via a message. In Scene 2, each racer arrives at a different time — the first to arrive (fastest racer) grows big, the others stay small. You control who wins by choosing the Move block values.
Character A
🐢 Racer 1 (slow)
Move Right = 3. Arrives last. Stays small in Scene 2.
Character B
🐇 Racer 2 (fast)
Move Right = 7. Arrives first. Grows in Scene 2.
Character C
🐸 Racer 3 (medium)
Move Right = 5. Arrives second. Stays small.
Character D
🏁 Referee
Sends the start message. Doesn't move.
📋 Build Instructions
1
Place all 4 characters in Scene 1. Line up the 3 racers at the left edge. Put the referee at the top centre. Set all racers facing right.
2
Referee script — sends the "GO" signal after a short delay:
Referee – Scene 1
▶ Green Flag ⏱ Wait 2 📨 Send Message (Yellow) ⬛ End
3
Each racer waits for the message, then runs. Racer 2 is fastest (bigger Move number):
Racer 2 (fast) – Scene 1
📬 Receive Message (Yellow) → Move Right 7 🎬 Change Scene → ⬛ End
Repeat for Racer 1 (Move Right 3) and Racer 3 (Move Right 5).
4
In Scene 2, add all 3 racers. Racer 2 (the winner) has this script:
Racer 2 – Scene 2
▶ Scene Begin ⬆ Grow ⬆ Grow ⬆ Grow ⬛ End
Racer 1 and 3 just have a ⬛ End block — they stay small.
5
Test your project. Change the Move numbers to swap who wins. Try making Racer 1 the winner by giving it Move Right = 8 and reducing the others.
⚠️
Important: All 3 racers must receive the same colour message from the referee. If they're listening for different colours, they won't all start at the same time.
💡
Design question: Could you make a race with a looped Move block (e.g., Repeat 4 × Move Right 2) instead of one large move? Would the result look different? Try it!
⭐
Extension: Add a countdown in Scene 1. Use 3 separate scripts on the referee: one that sends a message at Wait 0, one at Wait 1, one at Wait 2 — each making a different visual change (e.g. growing, shrinking). The 3rd message is the real GO signal for the racers.
Exercise 03
03

The Hiding Game

Hide/Show · Tap events · Message chains · Multi-scene logic
Advanced

🎬 Scene Plan

Scene 1
The Room
Hider runs and becomes invisible. Seeker searches by tapping.
→
Scene 2
Found!
Both characters appear. Hider grows, Seeker jumps.
🎯
Goal: The Hider moves to a corner and becomes invisible (shrink to nothing). The Seeker walks around. When the player taps the Seeker, it triggers a message that makes the Hider reappear (grow back). Then both characters do a reaction and the scene changes. You will use tap-on-character events for the first time to trigger a message chain.
Character A
🙈 The Hider
Runs to corner → shrinks to invisible → reappears when found.
Character B
🔍 The Seeker
Walks around searching. When tapped by player, sends "found!" message.
📋 Build Instructions
1
Hider – Scene 1 (hiding script): Run to a corner and shrink 5 times until invisible. Add waits to make it look gradual.
Hider – Scene 1
▶ Green Flag → Move Right 5 ↑ Move Up 3
⏱ Wait 1 ⬇ Shrink ⏱ Wait 1 ⬇ Shrink ⏱ Wait 1 ⬇ Shrink ⏱ Wait 1 ⬇ Shrink ⏱ Wait 1 ⬇ Shrink ⬛ End
2
Hider – Scene 1 (reappear script): The Hider also needs a second script — one that listens for a "found" message and makes it grow back.
Hider – Scene 1 (2nd script)
📬 Receive Message (Red)
⬆ Grow ⏱ Wait 1 ⬆ Grow ⏱ Wait 1 ⬆ Grow ⏱ Wait 1 ⬆ Grow ⏱ Wait 1 ⬆ Grow
📨 Send Message (Blue) ⬛ End
3
Seeker – Scene 1 (searching script): Walk left and right using a loop to simulate searching, then stop.
Seeker – Scene 1
▶ Green Flag ⏱ Wait 3
🔁 Repeat 3 [ → Move Right 2 ⏱ Wait 1 ← Move Left 2 ⏱ Wait 1 ]
⬛ End
4
Seeker – Scene 1 (tap event script): This is a separate script on the Seeker. Use the 👆 Tap on Character trigger. When the player taps the Seeker, it sends the "found" message.
Seeker – Scene 1 (2nd script)
👆 Tap on Character 📨 Send Message (Red) ⬛ End
5
Seeker – Scene 1 (react script): When the Seeker receives the Blue message (sent by Hider after reappearing), it jumps up and changes scene.
Seeker – Scene 1 (3rd script)
📬 Receive Message (Blue) ↑ Move Up 3 ⏱ Wait 1 ↓ Move Down 3 🎬 Change Scene → ⬛ End
6
Scene 2: Place both characters. Both use ▶ Scene Begin. Hider grows big (victory!), Seeker does a spin loop (Turn Right × 4 inside a Repeat 2).
⚠️
The message chain matters: The order is — Seeker gets tapped → sends Red → Hider receives Red, grows, sends Blue → Seeker receives Blue, jumps, changes scene. If any link in this chain is wrong, the whole sequence breaks. Draw it out before building!
💡
Multiple scripts on one character: In ScratchJr, one character can have several independent scripts at the same time. The hiding script and the reappear script on the Hider run separately — one starts at the green flag, the other waits for a message. This is normal and expected!
⭐
Extension: Add a third hiding spot. Place a second hidden character in Scene 1. The player must tap the Seeker twice — each tap sends a different colour message revealing a different character. Only when both are found does the scene change. How would you change the message chain to make this work?

🧠 Discussion Questions

Q1
In Exercise 1, why do you need a Wait block between each Shrink block? What would happen without them?
Q2
In Exercise 2, the 3 racers all receive the same Yellow message. Could you give them different Wait blocks before their Move block to stagger their starts? How would that change the race?
Q3
In Exercise 3, the Hider has two separate scripts. Why can't we put everything in a single script? What would happen if we tried?
Q4
In all three exercises, the scene change happens at the end of a script. What would happen if you put the scene change block at the beginning instead?
