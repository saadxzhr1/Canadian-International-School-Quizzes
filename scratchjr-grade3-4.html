Three structured projects that build on what you already know. Each exercise introduces a new problem to solve using scenes, timing, size, messages, and multi-character coordination.

📚 Grade 3–4
⏱ ~75–90 min total
🎬 3 projects
🚫 No sound blocks used

✅ You already know how to use

🎨 ScratchJr Block Colour Reference

Yellow Triggers, Receive Message, Send Message Blue Motion (Move, Turn, Hop) Purple Looks (Grow, Shrink, Hide, Show) Orange Control (Wait, Repeat, Go Fast/Slow) Red End block and Change Page only
Exercise 01
01

The Shrinking Escape

Size control · Scene transitions · Precise timing
Intermediate

🎬 Scene Plan

Scene 1
The Forest
Character shrinks gradually → scene changes
→
Scene 2
Spooky Forest
Character grows back to full size → walks to centre → scene changes
→
Scene 3
Safe Place
Character celebrates with a hop loop
🎯
Goal: A character shrinks small enough to "slip through" to a new scene, reappears tiny, grows back to full size, then celebrates. The challenge is controlling timing so the shrinking finishes before the scene changes, and the growing looks smooth on the other side.
Character
Hero — choose any ScratchJr character (e.g. Tic, Mia, Cat)
Main character. Place at left of Scene 1. Add the same character to Scenes 2 and 3.
📋 Build Instructions
1
Create 3 scenes/pages. Choose a different background for each. Add your character to all three scenes, positioned where you want them to appear.
2
Scene 1 — two scripts on the Hero, both starting with Green Flag. In ScratchJr, two scripts with the same trigger on the same character run at the same time. Use this to make the hero move right while shrinking simultaneously.
Hero – Scene 1 — Script 1 (movement)
▶ Green Flag → Move Right 1 → Move Right 1 → Move Right 1 → Move Right 1 ⬛ End
Hero – Scene 1 — Script 2 (shrinking + exit)
▶ Green Flag ⏱ Wait 1 ⬇ Shrink ⏱ Wait 1 ⬇ Shrink ⏱ Wait 1 ⬇ Shrink ⏱ Wait 1 ⬇ Shrink 👁 Hide 🎬 Change Page →
The hero moves right across the scene while getting smaller at the same time. The 👁 Hide block at the end makes the character truly vanish before the page changes.
3
Scene 2 script: Use the Start on Page trigger (orange page icon — yellow trigger block). Grow 4 times with waits, then walk right, then change page.
Hero – Scene 2
▶ Start on Page
⬆ Grow ⏱ Wait 1 ⬆ Grow ⏱ Wait 1 ⬆ Grow ⏱ Wait 1 ⬆ Grow
→ Move Right 3 🎬 Change Page →
4
Scene 3 script: Use ▶ Start on Page. Add a 🔁 Repeat 3 loop with ↑ Hop inside it to make the character jump 3 times in celebration. End with ⬛ End.
💡
Think about it: You wrote 4 separate Shrink blocks one after another. Look at your script — do you see a pattern repeating? In ScratchJr you can replace repeated blocks with a Repeat loop: one Shrink block inside a Repeat 4 does exactly the same thing with far fewer blocks. Try rewriting Script 2 using a loop. Can you do the same for the Grow blocks in Scene 2?
⭐
Extension: Add a Monster character (from the ScratchJr library) in Scene 1. It starts far left and walks toward the hero's starting position. Your goal: the Monster arrives exactly when the hero's Hide block fires and truly vanishes — not just small, but gone. Count the number of Move blocks the monster needs by measuring the distance from its start to the hero's start position. Use Wait blocks to align the timing.
Exercise 02
02

The Synchronized Race

Messages · Multi-character coordination · Staggered timing
Intermediate+

🎬 Scene Plan

Scene 1
The Starting Line
3 racers wait for message signal, then run at different speeds
→
Scene 2
The Finish Line
Winner grows big — others stay at original size
🎯
Goal: Three racers start at different speeds (controlled by different numbers in their Move blocks). A referee sends the GO signal via a message. The fastest racer triggers the scene change first — in Scene 2, it grows big. You control who wins by choosing the Move block values.
Racer 1 — Slow
e.g. Duck
Move Right = 3. Arrives last. No script in Scene 2.
Racer 2 — Fast
e.g. Cat
Move Right = 7. Arrives first. Grows in Scene 2.
Racer 3 — Medium
e.g. Dog
Move Right = 5. Arrives second. No script in Scene 2.
Referee
e.g. Parrot or Bear
Sends the start message. Stays in place. Only in Scene 1.
📋 Build Instructions
1
Place all 4 characters in Scene 1. Line up the 3 racers at the left edge. Put the referee at the top. Make sure all racers are facing right.
2
Referee script — waits 2 seconds then sends the GO signal. Send Message is a yellow block — add a red End block after it to close the script.
Referee – Scene 1
▶ Green Flag ⏱ Wait 2 📨 Send Message (Yellow) ⬛ End
3
Each racer listens for the yellow message and runs. The Receive Message block is yellow — it's a trigger. The fastest racer (Move Right 7) reaches the right side first and changes the page for everyone.
Racer 2 (fast – Cat) – Scene 1
📬 Receive Message (Yellow) → Move Right 7 🎬 Change Page →
Build the same script for Racer 1 (→ Move Right 3) and Racer 3 (→ Move Right 5).
4
In Scene 2, add all 3 racers. Only Racer 2 (the winner) gets a script — it grows to show it won. Racers 1 and 3 need no script at all in Scene 2; they will simply appear at their normal size.
Racer 2 (Cat) – Scene 2
▶ Start on Page ⬆ Grow ⬆ Grow ⬆ Grow ⬛ End
5
Test your project. Change the Move numbers to swap who wins. Try giving Racer 1 a Move Right of 8 to make it the winner.
⚠️
Important: All 3 racers must listen for the same colour message. If they have different colours, they won't all start at the same time.
⚠️
Change Page is red: The Change Page block ends the script automatically. Do not add an End block after it.
💡
Design question: Instead of one big Move Right block, could you use a Repeat loop with a smaller Move Right inside? (e.g. Repeat 4 × Move Right 2). Would the race look different? Try it!
⭐
Extension: Make the referee do a visual countdown. Add 3 scripts to the referee — each starts with a different Wait time (1, 2, 3) and each sends a different message that makes one of the racers briefly grow and shrink. The 4th message (after Wait 4) is the real GO signal.
Exercise 03
03

The Hiding Game

Hide · Show · Tap trigger · One message · Two scripts on one character
Intermediate+

🎬 Scene Plan

Scene 1
The Room
Hider walks to corner and hides. Seeker walks back and forth. Player taps Seeker to reveal the Hider.
→
Scene 2
Found!
Hider appears, grows, and hops to celebrate.
🎯
Goal: The Hider walks to a corner and uses the Hide block to become invisible. When the player taps the Seeker, it sends one message. The Hider receives that message, reappears with the Show block, grows, then changes page. New idea: one character can have two scripts — one that runs at the start, and one that waits for a message.
Character A — The Hider
e.g. Dinosaur or Monster
Script 1: walks to corner + hides. Script 2: receives message, shows, grows, changes page.
Character B — The Seeker
e.g. Tic or Mia
Script 1: walks left and right (searching). Script 2: when tapped, sends the message.
📋 Build Instructions
1
Hider – Script 1 (hide): Walk to the right side of the scene, wait a moment, then hide.
Hider – Scene 1 (Script 1)
▶ Green Flag → Move Right 5 ⏱ Wait 1 👁 Hide ⬛ End
2
Hider – Script 2 (reappear): A separate script on the same Hider. Waits for the message from the Seeker, shows itself, grows, then changes page.
Hider – Scene 1 (Script 2)
📬 Receive Message (Yellow) 👁 Show 🔁 Repeat 3
⬆ Grow ⏱ Wait 1 ← inside the loop
🎬 Change Page →
3
Seeker – Script 1 (searching): Waits, then walks back and forth using a Repeat loop.
Seeker – Scene 1 (Script 1)
▶ Green Flag ⏱ Wait 2 🔁 Repeat 3
→ Move Right 2 ⏱ Wait 1 ← Move Left 2 ⏱ Wait 1 ← inside the loop
⬛ End
4
Seeker – Script 2 (tap): A separate second script. When the player taps the Seeker, it sends the message that wakes up the Hider's Script 2.
Seeker – Scene 1 (Script 2)
👆 Tap on Character 📨 Send Message (Yellow) ⬛ End
5
Scene 2: Add the Hider. Use ▶ Start on Page. Make it hop 3 times using a 🔁 Repeat 3 loop with ↑ Hop inside, then ⬛ End.
⚠️
Two scripts, one character: The Hider's Script 1 (Green Flag) and Script 2 (Receive Message) are two completely separate scripts. Script 1 runs at the start. Script 2 sits and waits silently until it receives the message — only then does it activate. This is the key idea of this exercise.
💡
Notice the flow: Tap Seeker → Seeker sends Yellow message → Hider wakes up, shows, grows, changes page. Only one message colour is used. Draw this flow on paper before you build, and check each step works before going to the next.
⭐
Extension: In Scene 2, the Seeker is not there yet. Add the Seeker to Scene 2 with its own ▶ Start on Page script. Make it hop and then grow to celebrate finding the Hider. Both characters should animate when the scene starts.

🧠 Discussion Questions

Q1
In Exercise 1, the Hero uses two separate scripts that both start with the Green Flag. What happens when two scripts on the same character share the same trigger? Could you combine them into one single script and still make the hero move and shrink at the same time? Why or why not?
Q2
In Exercise 2, the 3 racers all receive the same Yellow message. Could you add a Wait block inside each racer's script (before the Move block) to stagger their starts? How would that change the race?
Q3
In Exercise 3, the Hider has two separate scripts. Why can't we put everything in one single script? What is the problem?
Q4
In Exercise 3, tapping the Seeker sends a message to the Hider. What would happen if the player taps the Seeker before the Hider has finished walking and hiding? Would it still work? Try it and explain what you see.
